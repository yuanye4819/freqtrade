# TrendGuard 趋势守护策略 — 新手友好说明文档

## 一、这个策略是干什么的？

这是一个 **加密货币期货自动交易策略**，在 Freqtrade 平台上运行。  
它的核心思想只有八个字：**顺势而为，回调入场**。

用大白话说：  
> "只有在市场上涨趋势中，趁价格短暂回调下跌时买入，然后用多种保护机制防止亏太多。"

---

## 二、和原来的策略比，改了什么？

原来的 `MyAwesomeStrategy` 策略用的是最简单的 RSI 超买超卖：

```
买入条件：RSI 从 30 以下涨上来  → 就买
卖出条件：RSI 从 70 以上跌下去  → 就卖
止损：   亏 10% 才砍
```

**问题出在哪？**

| 问题 | 解释 |
|------|------|
| ❌ 没有趋势判断 | 下降趋势中 RSI 也会超卖，但那是无底洞 |
| ❌ 止损太宽 | 亏 10% 才止损，但止盈才 1%，亏一笔要赚十笔 |
| ❌ 出场太随意 | RSI 一过热就跑，经常在牛市中途被踢下车 |

结果：84% 胜率但总账 **亏了 51%**，因为每次止损亏 10%，赚只赚 1%。

---

## 三、新策略怎么工作？

### 🟢 入场（买入条件）—— 六个条件全部满足才买

```
条件1: 价格高于 200 周期均线 (EMA200)   → "大趋势向上"
条件2: RSI 从超卖区（28以下）涨上来    → "回调到位"
条件3: 价格在布林带下轨附近              → "足够便宜"
条件4: 成交量大于平均量                   → "放量反弹有力度"
条件5: ADX 趋势强度 > 20                 → "不是横盘震荡"
条件6: 成交量 > 0                        → "K线有效"
```

> 📌 **通俗理解**：就像在大牛市里，等价格跌到"打折区"、有人开始抄底放量的时候才跟着买。

### 🔴 出场（卖出条件）—— 三层保护

| 保护层 | 触发条件 | 例子 |
|--------|----------|------|
| **ROI 止盈** | 持仓时间越久，要求收益越高 | 30分钟+1%、1小时+2%、2小时+3% |
| **追踪止损** | 盈利超 1.5% 后启动，回撤 2% 就卖 | 涨到 +3% 后回落 → +1% 时卖掉 |
| **硬止损** | 亏损达到 4% | 买错了，最多亏 4% 就认输 |

另外还有 **动态保护止损**：盈利越多，止损线抬得越高。

| 盈利 | 止损线 |
|------|--------|
| < 1% | -4%（默认） |
| 1-2% | -1.5% |
| 2-3% | 保本（开仓价） |
| > 3% | 锁定 +1% 利润 |
| > 5% | 锁定 +3% 利润 |

> 📌 **通俗理解**：赚了钱就不断把"止盈线"往上提，确保盈利不会全吐回去。

## 四、回测结果对比

> 回测区间：2025年1月 ~ 2026年7月（18个月）  
> 交易对：BTC/USDT、ETH/USDT（永续合约）

| 指标 | 原始策略 | TrendGuard | 改善 |
|------|----------|------------|------|
| 总盈亏 | **-51.11%** | **-3.74%** | ✅ 减少 93% |
| 交易笔数 | 487 | 94 | 少而精 |
| 胜率 | 84.6% | 60.6% | 高了不如对 |
| 最大回撤 | 52.94% | **6.95%** | ✅ 减少 87% |
| 夏普比率 | -2.03 | **-0.18** | 接近盈亏平衡 |
| 硬止损次数 | 60 | 20 | 少亏 67% |
| 每笔平均盈利 | +1.05% | +0.98% | 差不多 |
| 每笔平均亏损 | -10.1% | -4.1% | ✅ 少亏 60% |

**核心结论**：  
- 原来靠大量小赚 + 偶尔巨亏 = 总账巨亏 ❌  
- 现在靠精选入场 + 严格止损 = 亏损大幅缩小 ✅  
- 虽然还没盈利，但已经从"注定亏光"变成"接近打平"

## 五、策略参数详解

```python
# ========== 止盈 ==========
minimal_roi = {
    "120": 0.03,   # 持仓 2 小时 → 目标赚 3%
    "60":  0.02,   # 持仓 1 小时 → 目标赚 2%
    "30":  0.01,   # 持仓 30 分钟 → 目标赚 1%
    "0":   0.005,  # 无条件 → 目标赚 0.5%（防止无限持仓）
}

# ========== 止损 ==========
stoploss = -0.04  # 死线：最多亏 4%

# ========== 追踪止损 ==========
trailing_stop = True
trailing_stop_positive = 0.015       # 赚 1.5% 后才启动追踪
trailing_stop_positive_offset = 0.02 # 从最高点回撤 2% 触发

# ========== 入场参数（可超参优化） ==========
buy_rsi  = IntParameter(20, 40, default=28)   # RSI 低于 28 超卖 → 反弹买入
exit_rsi = IntParameter(65, 85, default=75)   # （此策略中出场不依赖此参数）
```

## 六、如何运行这个策略

### 1. 回测

```powershell
cd D:\freqtrade

# 完整回测
.\freqtrade-env\Scripts\freqtrade.exe backtesting `
  --config config.json `
  --strategy TrendGuardStrategy `
  --timerange 20250101-

# 只看最近一个月
.\freqtrade-env\Scripts\freqtrade.exe backtesting `
  --config config.json `
  --strategy TrendGuardStrategy `
  --timerange 20250601-20250701
```

### 2. 模拟交易（网页上跑）

```powershell
# 先改 config.json 里的 strategy 名
# "strategy": "TrendGuardStrategy"

.\freqtrade-env\Scripts\freqtrade.exe trade --config config.json
```

### 3. 超参优化（自动找最佳 RSI 阈值）

```powershell
.\freqtrade-env\Scripts\freqtrade.exe hyperopt `
  --config config.json `
  --strategy TrendGuardStrategy `
  --timerange 20250101-20250601 `
  --spaces buy `
  --epochs 200
```

## 七、如何继续改进

当前策略亏损主要是因为 **入场后涨得不够多**（平均只赚 1%）。可以尝试：

| 方向 | 做法 |
|------|------|
| 🎯 **提高入场质量** | 加入更多确认指标（如 OBV 量价背离、MACD 金叉） |
| 📈 **扩大盈利** | 放宽 ROI 表（比如 60分钟要 3%，不是 2%） |
| 🛡️ **收紧止损** | 把 4% 降到 3%，用更敏感的指标提前止损 |
| 🔄 **自适应参数** | 牛市用宽松、熊市用严格（根据市场状态动态切换） |
| 🤖 **超参优化** | 运行 hyperopt 让机器自动找最佳参数组合 |
